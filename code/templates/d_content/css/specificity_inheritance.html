Cascade, Specificity and Inheritance

For inheritance, some CSS properties inherit values set on the parent element by default and some do not. This might cause behaviour that you do not expect.
When you have rules conflicting with each other, cascade and specificity are mechanisms that control which rule applies. Therefore by understanding these 3 concepts, you will be able to prevent or fix unwanted behaviour.

Inheritance

The easiest way to understand inheritance is to see it in action.

.list {
color: rebeccapurple;
border: 2px solid #ccc;
padding: 1em;
}
.inner {
color: black;
font-weight: bold;
}

<ul class="list">
<li>One</li>
<li>Two
<ul>
<li>Two One</li>
<li>Two Two</li>
</ul>
</li>
<li>Three
<ul class=inner>
<li>Three One
<ul>
<li>Three One One</li>
<li>Three One Two</li>
</ul>
</li>
<li>Three Two</li>
</ul>
</li>
<li>Four</li>
</ul>

The colour has been applied to all direct and indirect children, the first layer of the list and also the nested list in "Two".
For the nested list in "Three", since the `.inner` rule has with the same property `color` defined, its children inherit `black` as the text colour instead of `rebeccapurple`.
Note that not all properties are inherited. Things like widths, margins, padding and borders were not inherited. It would be extremely annoying if these properties were inherited by the children as all item in the list would get a border.
Properties which are inherited or not can mostly be determined by common sense.

All CSS property accepts these values for controlling inheritance.

  • inherit: Set the property value to be as the same as the parent element.
  • initial: Set the property value to the initial value (default) of that property.
  • unset: Resets the property to its natural value. If a property naturally inherits a value, it will act like `inherit`, otherwise, it acts like `initial`.
Try out these value on the example above and see what happens.

Cascade

At a very simple level, stylesheets cascade means that the order of CSS rules matter. When two rules that have equal specificity, the one that is defined last in the CSS is the rule that applies. For example:

h1 {
color: blue
}
h1 {
color:green;
}

Specificity

The browser decides which rule will applies if multiple rules have different selectors, but would still apply to the same element. Basically a selector that is more specific will be selected compared to a less specific element. For example:

.a_class {
color: blueviolet;
}
p {
color: orange;
}

<p class="a_class"> A paragraph with class.</p>

Since both rules applies to the HTML snippet above and with the same property `color`, the browser needs to determine which to select. In this case, since a class selector is more specific than a type selector, `.a_class` rule will be applied and the colour of the text will be `blueviolet` instead of `orange`. NOTE: The entire rule were not overwritten, only the properties which are the same. If a less specific selector contains properties which are not present in a more specific selector, the non-conflicting properties in the less specific rule still applies. The browser determines specificity by calculating scores for specificity. If you would like to learn about how that is done, you can refer to this explanation by MDN.
