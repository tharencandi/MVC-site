How the Web Works

At a high level, the web is simply made up of clients and servers requesting and sending information.
A client is a device and its web accesing software such as a browser that is connected to the internet. The client sends requests to another device seeking a service or information. On the other hand, a server is a dedicated device that stores content and/or functionality, which when requested can be downloaded from the server.

Important Terminology

  • IP Address: IP address, as known as Internet Protocol Address, is a unique identifier for a device. IP addresses consists of four numbers from 0-255 divided by decimal points. For example, 168.212.226.204 is an IP address.
  • ISP: Internet Service Provider, as known as ISP, is a company providing internet services. They give user access to internet by different cables (e.g. Fibre-optics).
  • DNS: Domain Name System (DNS) is a database including the domain names and IP addresses of computers. Users can access www.google.com instead of its IP address by DNS.
  • Domain Name: Domain Name (e.g. www.google.com) is used to access the website and it has at least one corresponding IP address. Browser uses DNS to find the IP address of the domain name typed in.
  • TCP/IP: Transmission Control Protocol/Internet Protocol is a protocol for transmitting data through internet.
  • Port Number: A 16 bits integer associated with an IP address is a port number. Port number is a identification of the port on a server.
  • Host: A host is a device with one unique IP address connected to the internet. For example, a web server is one type of host which handles a web page. Besides, host is also the name of organization providing hosting services.
  • HTTP: Hyper-text Transfer Protocol, as known as HTTP, is a protocol used to communicate between web browsers and web servers.

The Process of Getting Things Displayed on Your Screen

  1. Type the URL into the browser.
  2. After you type the URL into the browser, The browser will parse the URL to get information (protocol, host name, and file name). If there is no file name, then the browser assumes to go to the main page.
  3. Then the browser and the ISP will communicate each other, also involving other services with DNS search to get the IP address of the website.
  4. The web browser then will take the IP address and the port number in URL through TCP. After this, the web browser and web server are connected.
  5. The web browser and web server then communicate each other for HTML resources.
  6. The web browser renders all the code in the HTML resources on the screen.
